# 채널톡 공식 용어집 (Channel Talk Glossary)

이 문서는 **장애 공지 작성 시 사용해야 할 채널톡 공식 용어**를 정리한 참고 자료다.

**출처**: https://docs.channel.io/help/ko (채널톡 공식 사용자 가이드)

공지를 쓸 때는 이 용어집에 있는 **정식 표기**를 반드시 따른다. 내부에서 쓰이는 줄임말, 영문 표기, 소문자 표기, 옛 이름은 모두 공식 표기로 변환한다.

---

## 1. AI 관련 용어 (가장 자주 등장)

| 정식 표기 | 설명 | 흔한 오표기 / 변환 대상 |
|---|---|---|
| **ALF** | 채널톡의 AI 에이전트. 반드시 **전부 대문자**로 표기. | "알프", "Alf", "alf" → **ALF** |
| **고객 ALF** | 고객 문의를 AI가 자동 처리하도록 돕는 상담 자동화 기능 (v2). | "고객 알프", "customer ALF" |
| **팀 ALF** | 팀 멤버가 업무 중 궁금한 점을 물어볼 수 있는 ALF. 유저챗/팀챗에서 호출 가능. | "내부 알프", "team ALF" |
| **ALF 전화 상담** | 상담원 대신 ALF가 전화 상담을 응대하는 기능 (Beta). | "전화 알프", "ALF phone" |
| **AI 부가기능** | 상담 요약 등 상담 후처리를 돕는 AI 기능. | "AI 보조기능", "AI 서포트" |

### ALF를 구성하는 요소 (공지에 그대로 써도 되는 공식 용어)

| 정식 표기 | 설명 |
|---|---|
| **규칙** | ALF가 따라야 할 지시문(프롬프트). 팀 멤버가 미리 작성. |
| **지식** | ALF가 참조하는 자료(문서/파일). |
| **태스크** | ALF가 단순 응답을 넘어 실제 업무까지 처리하는 자동화 기능. |
| **배포 설정** | ALF 프로필·규칙·지식·태스크를 모아 관리하는 메뉴. |
| **ALF 대화 목록** | 고객 ALF가 응대한 상담 목록. |

**중요**: "태스크가 동작한 유저챗"은 "태스크(ALF의 업무 자동화 기능)를 트리거한 유저챗"을 뜻하는 공식 표현이다. 장애 영향 범위가 태스크 실행에 걸린 경우 이 표현을 그대로 쓸 수 있다.

## 2. 메신저 / 상담 관련 용어

| 정식 표기 | 설명 |
|---|---|
| **팀 메신저** | 팀원끼리 사용하는 메신저 |
| **고객 메신저** | 고객과 대화하는 메신저 |
| **유저챗** | 고객과의 대화 단위 (한 건의 상담) |
| **팀챗** | 팀원 간 대화 단위 |
| **채널** | 비즈니스가 운영하는 채널톡 공간 (흔히 "우리 채널", "모든 채널" 등) |

## 공지 작성 시 체크리스트

공지문을 완성하기 직전에 이 체크리스트를 돌린다.

- [ ] "알프"가 어디에도 없다. 전부 **ALF**로 바뀌었다.
- [ ] 내부 모델명(예: "Claude sonnet 4.6", "GPT-4")이 공지 본문에 없다.
- [ ] 내부 컴포넌트명(예: "tool call", "invoker", "DB 커넥션 풀")이 공지 본문에 없다.
- [ ] "태스크", "규칙", "지식", "유저챗", "팀챗", "미트" 같은 채널톡 고유 용어는 **그대로** 쓴다. 굳이 풀어쓰지 않는다.
- [ ] 기능명 옆에 불필요한 영문 병기가 없다 (예: "유저챗(user chat)" ❌, "유저챗" ✅).

## 새로운 용어가 나왔을 때

이 용어집에 없는 채널톡 기능명이 원문에 등장하면:

1. 그 이름이 **사용자에게 노출되는 공식 기능명**인지 확인한다 (채널톡 가이드나 제품 UI에 등장하는가).
2. 노출되는 이름이면 그대로 쓴다.
3. 내부 코드네임이거나 개발팀 전용 용어면 **"시스템의 특정 구성 요소"** 또는 맥락에 맞는 일반 표현으로 바꾼다.
