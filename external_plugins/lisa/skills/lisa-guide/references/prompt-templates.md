# Lisa Prompt Templates

Copy and customize these templates for common tasks.

---

## Template: Feature Development

```markdown
# Mission
Build [feature name] for [project/system].

# Success Criteria
- [Specific endpoint/function works]
- [Tests pass]
- [Integration complete]

# Rules
- Commit after each component
- Write tests alongside code
- Output <promise>DONE</promise> when all criteria met
```

---

## Template: Code Migration

```markdown
# Mission
Migrate [source] from [old tech] to [new tech].

# Success Criteria
- All [files/modules] converted
- [Verification command] passes
- No [old tech] patterns remain

# Rules
- Convert one file at a time
- Commit after each conversion
- Run verification after each commit
- Output <promise>DONE</promise> when migration complete
```

## Template: Bug Fixing

```markdown
# Mission
Fix [specific bug/issue type] in [codebase area].

# Success Criteria
- [Command/test] passes without errors
- [Specific behavior] works correctly

# Rules
- Identify all instances first
- Fix one at a time
- Commit after each fix
- Output <promise>DONE</promise> when all fixed
```

## Template: Refactoring

```markdown
# Mission
Refactor [component/pattern] to use [new approach].

# Success Criteria
- All [old pattern] replaced with [new pattern]
- Tests still pass
- No functionality changed

# Rules
- Refactor one file at a time
- Run tests after each change
- Commit working changes only
- Output <promise>DONE</promise> when refactoring complete
```

## Template: Test Coverage

```markdown
# Mission
Add tests for [module/feature].

# Success Criteria
- [X]% coverage achieved
- All edge cases covered
- Tests are meaningful (not just line coverage)

# Rules
- Write tests for one function at a time
- Run tests after each addition
- Commit passing tests only
- Output <promise>DONE</promise> when coverage target met
```

## Template: Greenfield Project

```markdown
# Mission
Build [project type] that [main functionality].

# Phases
1. Project setup: Initialize with [stack]
2. Core: Implement [main features]
3. Polish: Add [secondary features]
4. Testing: Ensure all features work

# Rules
- Complete each phase in order
- Commit after each major component
- Test as you build
- Output <promise>DONE</promise> when project is complete and working
```
