# Phase Mapping Reference

## Feature-Dev Workflow Phases

### Phase 1: Discovery
**Purpose**: Understand requirements and constraints
**Pseudo-code applications**:
- `analyze_requirements(scope="...", constraints=[...])`
- `identify_dependencies(systems=[...], integrations=[...])`
- `assess_feasibility(technical="...", timeline="...")`

### Phase 2: Exploration
**Purpose**: Investigate solutions and design approach
**Pseudo-code applications**:
- `explore_solutions(approaches=[...], criteria=[...])`
- `prototype_design(architecture="...", components=[...])`
- `evaluate_tradeoffs(options=[...], priorities=[...])`

### Phase 3: Implementation
**Purpose**: Build the feature
**Pseudo-code applications**:
- `implement_feature(components=[...], order="...")`
- `write_tests(coverage="...", types=[...])`
- `refactor_code(targets=[...], patterns=[...])`

### Phase 4: Review & Refinement
**Purpose**: Validate and improve
**Pseudo-code applications**:
- `conduct_review(checklist=[...], stakeholders=[...])`
- `optimize_implementation(metrics=[...], targets=[...])`
- `document_decisions(context="...", rationale="...")`

## Phase Transition Triggers
- Discovery → Exploration: Requirements clarified
- Exploration → Implementation: Design approved
- Implementation → Review: Feature complete
- Review → Discovery: New iteration needed
